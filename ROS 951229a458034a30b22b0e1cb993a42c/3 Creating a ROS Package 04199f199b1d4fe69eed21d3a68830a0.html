3. Creating a ROS Package

A catkin package:

  1. contains a catkin compliant package.xml file. (The .xml file provides meta info about the package)
  1. contains a CMakeLists.txt which uses catkin. (If it's a catkin metapackage it must have the relevant boilerplate CMakeLists.txt file.)
  1. each package must have it's own folder. (No nested packages/sharing of the same directory)

The simplest catkin package may have a structure which looks like this:

my_package/
  CMakeLists.txt
  package.xml

It's recommended to work with catkin packages using a catkin workspace. But you can also build catkin packages standalone. A trivial workspace may look like this.

workspace_folder/        -- WORKSPACE
  src/                   -- SOURCE SPACE
    CMakeLists.txt       -- 'Toplevel' CMake file, provided by catkin
    package_1/
      CMakeLists.txt     -- CMakeLists.txt file for package_1
      package.xml        -- Package manifest for package_1
    ...
    package_n/
      CMakeLists.txt     -- CMakeLists.txt file for package_n
      package.xml        -- Package manifest for package_n

OK TIME FOR THE TUTORIAL -

  1. Create an empty catkin workspace (following tutorial 1)
  1. Creating a catkin package
    • Change directory to src file 
    $ cd ~/catkin_ws/src
    • Create a package
    ### Create a new package
catkin_create_pkg <package_name> [depend1] [depend2] [depend3]

Example:
$ catkin_create_pkg beginner_tutorials std_msgs rospy roscpp

# The package will depend on std_msgs, rospy and roscpp.
    • Every package should have at least one dependency. (There are six types of dependencies namely Build Dependencies, Build Export Dependencies, Execution Dependencies, Test Dependencies, Build Tool Dependencies, and Documentation Tool Dependencies)

    3. Sourcing the setup file

    $ . ~/catkin_ws/devel/setup.bash

    4. To view package dependencies use

    ### For all nested dependencies
$ rospack depends package_name

### For only the first-order direct dependencies
$ rospack depends1 package_name

    5. If you click into beginner_tutorials > src > you will see package.xml which is the metadata of your package. You can use 

    $ gedit package.xml

    to view the file in a text editor and make any changes.
