15. Recording and playing back data

  • A bag file (.bag) stores log data of every action performed in a period of time. The bag file can be run to reproduce all the actions of a node during the record time.
  • For a node like turtlesim, the mimic is not entirely accurate. The path tracked by turtlesim is very sensitive to small changes in timing in the system, and rosbag is limited in its ability to exactly duplicate the behavior of a running system in terms of when messages are recorded and processed by rosbag record, and when messages are produced and processed when using rosbag play. For nodes like turtlesim, where minor timing changes in when command messages are processed can subtly alter behavior, the user should not expect perfectly mimicked behavior.
  • To demonstrate how this works:
  1. Run a node.
# make a directory called bagfiles as good practice
$ mkdir ~/bagfiles
cd ~/bagfiles

3. Start recording (within the bagfiles directory)

$ rosbag record -a # -a indicates that all published topics should be accumulated in a bag file.

4. Stop recording by doing Ctrl-C

5. To see info about the bag file, go to the bagfiles folder and copy the name of the file. (should be a date)

$ rosbag info filename.bag

6. To play back the file:

$ rosbag play filename.bag

7. You can change the speed of the playback (changes the trajectory of the turtle as well, as an example):

$ rosbag play -r 2 filename.bag
# -r changes the rate. 
# 2 means twice the rate.

Recording a Subset of the Data

  • Maybe you don't wanna record every single topic that's currently running.
  • For example in the turtlesim there are these topics running:
/rosout
/turtle1/cmd_vel
/turtle1/color_sensor
/turtle1/pose
  • But you only want to record the velocity and the pose.
  • So you can run this:
$ rosbag record -O subset /turtle1/cmd_vel /turtle1/pose

# rosbag record will only subscribe to these 2 topics.
